# Dyson Maps Pipeline + Preview App

## Project Overview

This repository has two parts that work together:

1. Python pipeline
- Crawls Dyson Logos map pages (needs an update)
- Caches HTML
- Extracts image candidates and tags
- Downloads selected assets
- Generates web-ready previews and final JSON for the UI

2. React preview app (`map-preview-app`)
- Reads the generated JSON from `/data/download_queue_web.json`
- Displays preview images from `/data/maps_preview`
- Provides original download links from `/data/map_assets`

Generated files are written under `data/outputs` and then exposed to the frontend through `map-preview-app/public/data`.

---

## How To Run

### Prerequisites

- Python 3.11+
- Node.js 20+
- npm

### 1) Install Python dependencies

From the repository root:

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

### 2) Run the Python pipeline

Run all stages (incremental by default):

```powershell
.\.venv\Scripts\python.exe scripts/run_pipeline.py --config conf/default.json --verbose
```

Run a single stage (example: preview generation only):

```powershell
.\.venv\Scripts\python.exe scripts/run_pipeline.py --config conf/default.json --stage s04
```

Useful stage shortcuts:
- `s00` crawl
- `s01` fetch HTML
- `s02` extract image candidates
- `s03` download map assets
- `s04` generate previews + web JSON

### 3) Run the frontend locally

```powershell
cd map-preview-app
npm install
npm run dev
```

Open: `http://localhost:5173`

### 4) Optional: run frontend with Docker Compose

From repository root:

```powershell
docker compose up
```

This starts the Vite app in a container and mounts `data/outputs` into `map-preview-app/public/data`.

## Main Output Files

- `data/raw/download_queue.csv`: extracted candidates after crawl/parse stages
- `data/raw/download_queue_web.csv`: web-oriented CSV generated by S04
- `data/outputs/download_queue_web.json`: primary frontend payload
- `data/outputs/maps_preview`: generated preview images
- `data/outputs/map_assets`: downloaded original map assets

---

## Notes

- Docker is not required for the Python pipeline.
- The recommended workflow is: run pipeline locally with `.venv`, then run frontend locally or with Docker.
- Tag cleanup (including singular/plural fusion and canonicalization) is done in S04 before data reaches the app.
